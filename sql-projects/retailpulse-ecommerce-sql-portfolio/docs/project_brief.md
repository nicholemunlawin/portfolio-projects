# Project Brief

## Why I Built This

I wanted a SQL project that felt closer to everyday ecommerce analytics work than a single-table exercise. RetailPulse gives me enough tables to practice realistic joins, but it is still small enough to explain clearly in a portfolio walkthrough.

The dataset covers the full order flow:

- customer signup and acquisition
- product catalog and suppliers
- orders, payments, and line items
- fulfillment, warehouses, and inventory
- reviews, returns, and support tickets

## Analysis Plan

My first pass focuses on questions a retail team would probably ask in a weekly business review:

1. Revenue trend by month and channel
2. Best-selling products and categories
3. Estimated gross profit by category
4. Repeat purchase rate by acquisition channel
5. Customer lifetime value ranking
6. RFM-style customer segments
7. Delivery time by carrier and warehouse
8. Inventory below reorder level
9. Return rate by category
10. Support resolution time by topic

## SQL Skills Covered

- normalized table design
- primary and foreign keys
- indexes for common joins and filters
- multi-table joins
- CTEs
- window functions
- conditional aggregation
- date calculations
- basic data quality checks
